1. Core Philosophy of SMILE
SMILE (Syntax Memes In Lahori English) is a completely original mini programming language designed around Pakistani internet culture, Roman Urdu, and meme-based vocabulary. No keyword in SMILE copies any existing programming language.
The language philosophy follows three principles:
Originality — Every keyword is invented and culturally grounded in Roman Urdu.
Readability — The language reads like spoken Pakistani English / Urdu, making it memorable.
Consistency — Every rule is uniformly applied across all constructs.
2. Language Specification
All keywords, operators, and delimiters are original inventions. Standard keywords such as int, if, while, return, and print are forbidden in SMILE.

3. Token Identification
Every element in SMILE belongs to exactly one token category. The six categories are:
KEYWORD — reserved words that control language structure and logic
IDENTIFIER — programmer-defined variable and function names
OPERATOR — symbols/words that perform computations or comparisons
NUMBER — numeric literal values (integers and decimals)
DELIMITER — structural tokens that define statement and block boundaries
STRING — text literals enclosed in double quotes

5.4 Invalid Program 1 — Missing 'bas' Terminator
Statements are written without the required 'bas' terminator. The lexer cannot determine where one statement ends and the next begins.
~ Error: bas bhool gaye ~
shuru karo
  adad x rakho 10        <- bas missing!
  adad y rakho 20        <- bas missing!
  batao x joro y bas
khatam
faaaah: line 2 -> statement khatam nahi hua, 'bas' lagao yaar
faaaah: line 3 -> phir se wahi galti, 'bas' kahan hai bhai?
Total errors: 2 -- program nahi chalay ga
Reason: SMILE's grammar requires every statement to end with 'bas'. Without it, the lexer treats both statements as one continuous token stream, causing a parse error.
5.5 Invalid Program 2 — Identifier Starts with Number
Variable names begin with digits, violating the SMILE identifier rule: [a-zA-Z][a-zA-Z0-9_]*
~ Error: galat identifier naam ~
shuru karo
  adad 3score rakho 100 bas    <- invalid identifier!
  adad 99life rakho 3 bas      <- invalid identifier!
  batao 3score bas
khatam
faaaah: line 2 -> '3score' valid identifier nahi -- number se start nahi hota bhai
faaaah: line 3 -> '99life' bhi galat hai -- pehle letter, phir number
Total errors: 2 -- sahi naam rakho, yaar
Reason: SMILE identifier regex is [a-zA-Z][a-zA-Z0-9_]* — must start with a letter. '3score' starts with a digit so the lexer classifies it as a malformed NUMBER token, triggering a type error.
5.6 Invalid Program 3 — Block Never Closed
A 'shuru' block is opened for the if-condition but the matching 'banda' is never written, causing an unclosed block error.
~ Error: banda bhool gaye ~
shuru karo
  adad age rakho 17 bas
  agar age kam 18 shuru
    batao "Abhi chota hai" bas
    ~ banda yahan hona chahiye tha! ~
  batao "Program khatam" bas
khatam        <- banda missing for agar block!
faaaah: line 4 -> 'shuru' se block khola tha
faaaah: line 9 -> 'khatam' mil gaya lekin 'banda' nahi aaya -- block band karo pehle
Total errors: 1 -- structure theek karo bhai
Reason: Every 'shuru' must have a matching 'banda'. The parser tracks block depth on a stack. When 'khatam' arrives with an open block still on the stack, it throws an unclosed block error.
6. Design Choices & Justification
Every design decision in SMILE is intentional. Below are the key choices and their rationale for the viva.
6.1  Why 'bas' instead of ';' ?
'Bas' is Roman Urdu for 'that's it' or 'done'. When a Pakistani speaker finishes a sentence they say 'bas' — it naturally signals the end of a thought. This makes SMILE's statement terminator semantically meaningful rather than an arbitrary punctuation mark.
6.2  Why 'shuru / banda' for blocks?
'Shuru' means 'start' and 'banda' means 'closed/shut' in Roman Urdu. They form a clear oppositional pair that mirrors the opening and closing of curly braces { } in standard languages, but with human-readable meaning. A programmer can read the code aloud and immediately understand its structure.
6.3  Why 'faaaah' for errors?
'Faaaah' is Pakistani internet slang expressing frustration or failure — it is the natural reaction when something goes wrong. Using it as the error prefix makes compiler error messages culturally resonant, memorable, and actually informative in Roman Urdu. It also distinguishes SMILE error output from any standard compiler.
6.4  Why 'adad' for integers?
'Adad' is the actual Urdu word for number or digit. It was chosen because it is semantically accurate — unlike arbitrary keywords, it directly communicates the variable's purpose to any Roman Urdu speaker, keeping the language self-documenting.
6.5  Why 'shuru karo' as the program entry point?
'Shuru karo' means 'let us begin' in Urdu. Every SMILE program starts with this phrase — the programmer is literally telling the computer to start. It replaces main() with a natural language equivalent that any Urdu speaker intuitively understands.
6.6  Why word-based operators instead of symbols?
SMILE deliberately avoids operator symbols like +, -, >, == and replaces them with words (joro, ghata, zyada, barabar). This choice makes expressions readable as sentences. For example: 'agar marks zyada ya barabar 60' reads as 'if marks greater than or equal to 60' — no symbol decoding required.
6.7  Why Roman Urdu and not pure English?
Roman Urdu (Urdu written in English letters) is the dominant written form of informal Pakistani communication — used in WhatsApp, Twitter, memes, and everyday texting. SMILE's target audience is Pakistani students, so using their natural written voice makes the language immediately accessible, fun to write, and easy to remember.
